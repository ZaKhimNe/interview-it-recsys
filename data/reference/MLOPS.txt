# MLOPS — Reference Document (Source: GPT-4o)

## ML Pipeline Design

- **Training/Serving Skew**
  - **Definition**: The discrepancy between the data or environment used during model training and the data or environment during model serving.
  - **Mechanism**: Occurs when features, data distributions, or preprocessing steps differ between training and serving. This can lead to degraded model performance in production.
  - **Importance**: Ensures that the model performs consistently in production as it did during training. Skew can lead to incorrect predictions and business decisions.
  - **Common Confusions**: Many candidates confuse training/serving skew with concept drift. The key difference is that skew is about differences in data processing, while concept drift is about changes in data distribution over time.

## Experiment Tracking with MLflow

- **Runs, Params, Metrics, Artifacts**
  - **Definition**: MLflow tracks experiments by logging runs, parameters, metrics, and artifacts.
  - **Mechanism**: 
    - **Runs**: Instances of model training.
    - **Params**: Hyperparameters used in a run.
    - **Metrics**: Performance metrics like accuracy or loss.
    - **Artifacts**: Files like model binaries or plots.
  - **Importance**: Facilitates reproducibility and comparison of different model versions.
  - **Red Flags**: Failing to track all relevant parameters and metrics can lead to incomplete experiment analysis.

- **Model Registry**
  - **Definition**: A centralized repository to manage model versions.
  - **Mechanism**: Supports model lifecycle stages like staging, production, and archiving.
  - **Importance**: Ensures traceability and governance of models in production.
  - **Common Confusions**: Confusing model registry with artifact storage. The registry manages model lifecycle, while artifact storage is for files.

## Model Versioning

- **Definition**: The practice of managing multiple versions of a model.
- **Mechanism**: Involves tagging models with version numbers and metadata.
- **Importance**: Critical for rollback and comparison of model performance over time.
- **Red Flags**: Lack of version control can lead to deploying incorrect model versions.

## Deployment Patterns

- **REST API Inference**
  - **Definition**: Deploying models as RESTful services for real-time predictions.
  - **Mechanism**: Models are wrapped in a web server that handles HTTP requests.
  - **Importance**: Enables integration with other systems and applications.
  - **Complexity**: Latency and throughput considerations are crucial.

- **Batch Scoring**
  - **Definition**: Processing large datasets in batches for predictions.
  - **Mechanism**: Models are applied to datasets at scheduled intervals.
  - **Importance**: Efficient for non-time-sensitive applications.
  - **Complexity**: Requires handling of large data volumes.

- **Streaming Inference**
  - **Definition**: Real-time model predictions on streaming data.
  - **Mechanism**: Models are integrated with streaming platforms like Kafka.
  - **Importance**: Essential for applications requiring immediate insights.
  - **Complexity**: Requires low-latency processing and fault tolerance.

## Model Monitoring

- **Data Drift Detection**
  - **Definition**: Identifying changes in input data distribution over time.
  - **Mechanism**: Statistical tests like KS-test or monitoring feature distributions.
  - **Importance**: Ensures model predictions remain accurate.
  - **Thresholds**: p-value < 0.05 often used to detect significant drift.

- **Concept Drift**
  - **Definition**: Changes in the relationship between input data and target variable.
  - **Mechanism**: Monitored using performance metrics over time.
  - **Importance**: Critical for maintaining model relevance.
  - **Common Confusions**: Confused with data drift; concept drift affects model accuracy directly.

- **Performance Degradation Alerts**
  - **Definition**: Notifications triggered when model performance drops below a threshold.
  - **Mechanism**: Automated monitoring systems track metrics like accuracy or F1-score.
  - **Importance**: Allows timely intervention to retrain or update models.

## CI/CD for ML

- **Testing Data Pipelines**
  - **Definition**: Validating data processing steps in ML pipelines.
  - **Mechanism**: Unit tests and integration tests ensure data integrity.
  - **Importance**: Prevents data quality issues from affecting model performance.

- **Model Validation Gates**
  - **Definition**: Checkpoints in the CI/CD pipeline to ensure model quality.
  - **Mechanism**: Automated tests for metrics, bias, and fairness.
  - **Importance**: Ensures only high-quality models are deployed.

## Feature Store

- **Online vs Offline**
  - **Definition**: Systems for managing and serving features for ML models.
  - **Mechanism**: 
    - **Online**: Low-latency access for real-time predictions.
    - **Offline**: Batch processing for training and batch inference.
  - **Importance**: Ensures consistency and efficiency in feature usage.
  - **Common Confusions**: Confusing feature store with data warehouse; feature stores are optimized for ML use cases.

## A/B Testing Models

- **Definition**: Comparing two model versions to evaluate performance differences.
- **Mechanism**: Randomly splitting traffic between models and analyzing outcomes.
- **Importance**: Provides empirical evidence for model improvements.
- **Complexity**: Requires careful design to avoid biases and ensure statistical significance.

## SHAP Values

- **Definition**: A method to explain individual predictions by assigning feature importance.
- **Mechanism**: Based on cooperative game theory, calculates contribution of each feature.
- **Importance**: Enhances model interpretability and trust.
- **Complexity**: Computationally expensive for large models or datasets.

## LIME

- **Definition**: Local Interpretable Model-agnostic Explanations, a technique to explain model predictions.
- **Mechanism**: Perturbs input data and observes changes in predictions to approximate a local linear model.
- **Importance**: Provides insights into model behavior for specific instances.
- **Common Confusions**: Confused with SHAP; LIME is local and model-agnostic, while SHAP provides global explanations.

## Interviewer Red Flags

- Confusing training/serving skew with concept drift.
- Failing to track all relevant parameters and metrics in MLflow.
- Lack of understanding of the difference between online and offline feature stores.
