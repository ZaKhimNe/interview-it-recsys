## Missing Value Strategies

- **Mean/Median/Mode Imputation**
  - **Definition**: Replacing missing values with the mean, median, or mode of the available data.
  - **Mechanism**: 
    - Mean imputation replaces missing values with the average of non-missing values.
    - Median imputation uses the middle value, which is robust to outliers.
    - Mode imputation is used for categorical data, replacing missing values with the most frequent category.
  - **When/Why It Matters**: 
    - Mean is suitable for symmetric distributions without outliers.
    - Median is better for skewed distributions or when outliers are present.
    - Mode is essential for categorical features.
  - **Common Confusions**: Confusing mean with median imputation; mean is sensitive to outliers, while median is not.
  - **Interview-Critical Distinctions**: Recognizing when to use each method based on data distribution.

- **KNN Imputation**
  - **Definition**: Imputes missing values using the nearest neighbors in the feature space.
  - **Mechanism**: 
    - Identifies k-nearest neighbors based on a distance metric (e.g., Euclidean).
    - Imputes missing values using the mean (or median) of the neighbors.
  - **When/Why It Matters**: 
    - Effective when data points are similar to their neighbors.
    - Computationally expensive for large datasets.
  - **Interview-Critical Distinctions**: Understanding computational cost and when it is justified.

- **Indicator Variables**
  - **Definition**: Adding a binary variable to indicate the presence of missing values.
  - **Mechanism**: 
    - Creates a new feature where 1 indicates a missing value and 0 indicates a non-missing value.
  - **When/Why It Matters**: 
    - Useful when missingness itself is informative.
  - **Interview-Critical Distinctions**: Recognizing when missingness is not random and could impact model performance.

## Outlier Detection

- **IQR Method**
  - **Definition**: Uses the interquartile range to identify outliers.
  - **Mechanism**: 
    - Outliers are values below Q1 - 1.5*IQR or above Q3 + 1.5*IQR.
  - **When/Why It Matters**: 
    - Effective for skewed distributions.
  - **Common Confusions**: Misapplying IQR to non-continuous data.
  - **Interview-Critical Distinctions**: Correctly identifying the IQR range and its application.

- **Z-score**
  - **Definition**: Measures how many standard deviations a data point is from the mean.
  - **Mechanism**: 
    - Z = (X - μ) / σ
    - Outliers typically have |Z| > 3.
  - **When/Why It Matters**: 
    - Suitable for normally distributed data.
  - **Interview-Critical Distinctions**: Recognizing the assumption of normality.

- **Isolation Forest**
  - **Definition**: An ensemble method that isolates observations by randomly selecting a feature and a split value.
  - **Mechanism**: 
    - Constructs trees where anomalies are isolated closer to the root.
  - **When/Why It Matters**: 
    - Effective for high-dimensional data.
  - **Interview-Critical Distinctions**: Understanding its unsupervised nature and scalability.

## Feature Scaling

- **StandardScaler**
  - **Definition**: Scales features to have zero mean and unit variance.
  - **When/Why It Matters**: 
    - Essential for algorithms assuming normally distributed data (e.g., PCA, linear regression).
  - **Interview-Critical Distinctions**: Recognizing its impact on distance-based algorithms.

- **MinMaxScaler**
  - **Definition**: Scales features to a specified range, typically [0, 1].
  - **When/Why It Matters**: 
    - Useful when features have different units or scales.
  - **Interview-Critical Distinctions**: Understanding its sensitivity to outliers.

- **RobustScaler**
  - **Definition**: Scales features using statistics that are robust to outliers (median and IQR).
  - **When/Why It Matters**: 
    - Ideal for data with outliers.
  - **Interview-Critical Distinctions**: Knowing when to prefer it over StandardScaler or MinMaxScaler.

## Encoding

- **One-Hot Encoding**
  - **Definition**: Converts categorical variables into binary vectors.
  - **When/Why It Matters**: 
    - Necessary for algorithms that require numerical input.
  - **Common Confusions**: Misunderstanding the curse of dimensionality with high cardinality features.
  - **Interview-Critical Distinctions**: Recognizing its impact on model complexity.

- **Label Encoding**
  - **Definition**: Assigns a unique integer to each category.
  - **When/Why It Matters**: 
    - Suitable for ordinal data.
  - **Common Confusions**: Misapplying to nominal data, leading to unintended ordinal relationships.
  - **Interview-Critical Distinctions**: Understanding when label encoding introduces bias.

- **Target Encoding**
  - **Definition**: Replaces categories with the mean of the target variable for each category.
  - **Risks**: 
    - Can lead to overfitting, especially with small datasets.
  - **Interview-Critical Distinctions**: Recognizing the need for cross-validation or regularization.

## Feature Engineering

- **Definition**: The process of creating new features or modifying existing ones to improve model performance.
- **Mechanism**: 
  - Includes transformations, aggregations, and domain-specific knowledge.
- **When/Why It Matters**: 
  - Critical for enhancing model accuracy and interpretability.
- **Interview-Critical Distinctions**: Demonstrating creativity and domain knowledge in feature creation.

## Feature Selection

- **Filter Methods**
  - **Definition**: Select features based on statistical tests (e.g., chi-square, ANOVA).
  - **When/Why It Matters**: 
    - Fast and independent of the learning algorithm.
  - **Interview-Critical Distinctions**: Understanding its limitations with feature interactions.

- **Wrapper Methods**
  - **Definition**: Use a predictive model to evaluate feature subsets.
  - **When/Why It Matters**: 
    - More accurate but computationally expensive.
  - **Interview-Critical Distinctions**: Recognizing the trade-off between accuracy and computational cost.

- **Embedded Methods**
  - **Definition**: Perform feature selection as part of the model training process (e.g., LASSO).
  - **When/Why It Matters**: 
    - Balances accuracy and efficiency.
  - **Interview-Critical Distinctions**: Understanding how regularization impacts feature selection.

## Imbalanced Datasets

- **SMOTE (Synthetic Minority Over-sampling Technique)**
  - **Definition**: Generates synthetic samples for the minority class.
  - **When/Why It Matters**: 
    - Effective for balancing class distribution.
  - **Interview-Critical Distinctions**: Recognizing its impact on model variance.

- **ADASYN (Adaptive Synthetic Sampling)**
  - **Definition**: Similar to SMOTE but focuses on harder-to-learn examples.
  - **When/Why It Matters**: 
    - Addresses class imbalance more adaptively.
  - **Interview-Critical Distinctions**: Understanding its focus on difficult samples.

- **Class Weight**
  - **Definition**: Adjusts the cost function to penalize misclassification of minority class more heavily.
  - **When/Why It Matters**: 
    - Useful for algorithms that support class weights (e.g., SVM, logistic regression).
  - **Interview-Critical Distinctions**: Recognizing when it is preferable to resampling techniques.

- **Undersampling**
  - **Definition**: Reduces the size of the majority class.
  - **When/Why It Matters**: 
    - Simple and effective for small datasets.
  - **Interview-Critical Distinctions**: Understanding the risk of losing important information.
